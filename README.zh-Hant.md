# EvernightBoard 長夜看板

[English](README.md) | [简体中文](README.zh-Hans.md) | **繁體中文** | [日本語](README.ja.md)

**寄長夜之靈犀，化萬語為此間。**

EvernightBoard 是一款透過預設圖文，在觸控螢幕和語言交流都受限時為您延續溝通的展示輔助工具。

Android | iOS | Windows | macOS | Linux

## 應用場景

- 在使用手套等**觸控螢幕精準度極低**或**無法使用觸控螢幕**，同時言語交流受限的時候，使用預設的大型文字或圖片，快速向他人傳達資訊。
- 單純需要在螢幕上以大字顯示文字訊息或圖片，並能操作換頁的情境。

### 支援的換頁方式

- 導覽列切換螢幕分頁
- 點擊半邊螢幕切換螢幕分頁
  - 直向螢幕為上半部與下半部
  - 橫向螢幕為左半部與右半部

## 使用方法

### 準備工作

1. 啟動 App 後，可以看到空白內容和一個初始分頁（**新螢幕**）。 可以看到有兩個區域：
    1. **導覽列區域**：內含多個可自行修改和新增的按鈕，每個按鈕代表一個「**螢幕**」。
    2. **空白區域**：即目前「**螢幕**」，您可以指定它顯示一些文字還是圖片。
2. 按下**目前選取的**分頁（如果是第一次使用，那就是唯一的那個分頁），可以打開選單。 選單中包括以下項目：
    1. **導覽列分頁相關**（淡藍色）
        1. **側邊欄圖示**：可以更換側邊欄中目前項目的圖示，您可以從打開的圖示庫中選擇一個。
        2. **側邊欄標題**：圖示下方的標籤，不支援換行，並且內容建議盡可能簡短。
    2. **導覽列分頁順序調整**（綠色）
        1. **上移**：將這個圖示在導覽列中移動到前一個位置，如果已經在最前則自動移動到最後。
        2. **下移**：將這個圖示在導覽列中移動到後一個位置，如果已經在最後則自動移動到最前。
    3. **設定目前螢幕內容**（橘色）
        1. **設為文字**：目前螢幕將顯示一段您設定的文字，支援換行，文字將自動填滿並最大化顯示。
        2. **設為圖片**：目前螢幕將顯示一張您設定的圖片，注意圖片檔案不要太大以免浪費記憶體。
    4. **設定顏色**（粉紅色）：這裡調整的顏色對**位於目前螢幕**的導覽列也有效。
        1. **文字顏色**：設定目前螢幕的**文字模式**的文字顏色 和 目前螢幕的導覽列的文字顏色。
        2. **背景顏色**：設定目前螢幕的背景顏色 和 目前螢幕的導覽列的背景顏色，小心不要和文字設定為同樣的顏色。
    5. **新增或刪除螢幕**（藍色）
        1. **新增螢幕**：建立一個新螢幕，您可以點擊導覽列中新的螢幕按鈕切換到它，**再次點擊它**可以打開選單進行編輯。
        2. **複製螢幕**：複製目前螢幕到一個新的螢幕，包括文字/圖片/顏色。
    6. **刪除螢幕**（紅色）：將會移除目前螢幕，將遺失目前螢幕的所有內容。
    7. **應用程式設定**（灰色）：顯示更多功能。
        1. 翻頁互動
            1. **點擊半邊螢幕翻頁**開關：打開後可以按半邊螢幕切換螢幕（直向螢幕為上半部與下半部，橫向螢幕為左半部與右半部）
            2. **音量鍵翻頁**開關：使用裝置的音量按鈕來切換螢幕，適用於完全無法使用觸控螢幕的場合。
        2. 設定管理：分配**螢幕設定**和**資料設定**。
            1. **匯出設定**：將**螢幕設定**匯出到 JSON 檔案。 如果有圖片將會嵌入，導致匯出檔案顯著變大。
            2. **匯入設定**：從 JSON 檔案匯入**螢幕設定**，建議僅匯入相同版本 App 的設定。
            3. **恢復原廠設定**：清空所有**螢幕設定**和**軟體設定**。
        3. 關於
            1. **幫助和資訊**：打開關於視窗
                1. **使用說明**、**問題回報**、**原始碼**：打開瀏覽器造訪相關網頁。
                2. **檢視協議**：列出本軟體和本軟體使用的所有第三方函式庫的授權條款。
            2. **結束程式**：完全結束程式，釋放記憶體，不會在背景留存。

#### 其他建議

- 暫時停用手機的鎖定螢幕功能。
- 將應用程式圖示放在主畫面或易於啟動的位置。

### 開始使用

按照上面的準備工作準備好多個預設的圖片或文字之後，即可在以下場合輕鬆向別人告知預設的資訊。

1. 觸控螢幕精準度極低的場合：按半邊螢幕切換螢幕。
2. 完全無法使用觸控螢幕的場合：使用音量按鈕切換螢幕。

#### 螢幕設定範例

1. 「你好」
2. 「我可以和你合照嗎」
3. 「看那邊，他來拍照」
4. 「謝謝」
5. 「你很好看」
6. 「我想去那邊」
7. 「我想去和他合照」
8. 「抱歉，我沒有聽清楚」
9. 「我沒辦法說話」
10. 「交給我吧」
11. 「我想休息一會」
12. 「看一下群組裡的訊息」
13. 「我的相機要沒電了」
14. 「我的手機要沒電了」
15. (社群帳號 QR Code)

所以你猜這個應用程式最初設計是為了什麼情境的 `^_^`

## 編譯

### 環境要求

1. Flutter : 您可以在 `pubspec.yaml` 檔案的 `dependencies:flutter:` 處的註解看到最佳的 Flutter 版本。
2. 執行 `flutter doctor` 指令，根據提示完成各種設定。
3. `cd` 進入本專案所在的資料夾。

### 在 Windows 中編譯

- 編譯為 Windows 應用程式並執行: `build.bat` 。
- 編譯為 Android 應用程式並安裝: `build_apk.bat` 。

### 在 macOS 或 Linux 中編譯

- 編譯為 macOS 或 Linux 應用程式並執行: `./build` 。
- 編譯為 Android 應用程式並安裝: `./build_apk` 。

### 編譯為 macOS 或 iOS

1. 先嘗試編譯一次。
2. 使用 Xcode 打開 `macos` 或 `ios` 資料夾中的 `Runner.xcworkspace` 進行設定（例如憑證和描述檔）。
3. 再進行正式編譯。

### 偵錯

1. 執行 `flutter clean` 。
2. 執行 `flutter pub get` 下載所需第三方函式庫。
3. 執行 `generate_icons.bat` 或 `./generate_icons` 產生各種規格和樣式的應用程式圖示。
4. 執行 `dart.bat run flutter_iconpicker:generate_packs --packs material` 準備圖示資源。
5. 執行 `flutter run` 開始偵錯。
